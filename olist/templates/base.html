Welcome to Olist challenge

This platform it's a complete CRUD, with Back-end in Django rest framework and Front-end in AngularJs. More details you can find on README documentation.

All filters can read part of a text or number, ex: Product is 'Truck', you can write only 'Tru' (if more products starts with 'tru', it will be returned).

Details of how to fill an input are always in the input placeholder.

The filters are case insensitive (has no difference between 'a' and 'A'). All tabs has filters.

Normal filters (search bar) search the result in any field. Advanced filters give you the possibility to combine filters to get a result
